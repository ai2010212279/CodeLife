1.Java类的加载过程

```
第一步：加载阶段，类加载器通过类的全限定名获取到类的二进制文件流，并在方法区中生成一个java.lang.Class对象
第二步：验证阶段，虚拟机会对Class文件进行校验，主要是为了保证文件的格式符合JVM的安全要求。
第三步：准备阶段，这个阶段将正式为类变量分配内存，并设置类变量的初始值，这些变量需要的内存会在方法区中进行分配。
第四步：解析阶段，解析阶段是虚拟机将常量池内的符号引用替换为直接引用的过程。可以理解为，让类与静态变量发生绑定的过程。
第五步：初始化阶段，这个阶段其实就是静态变量的赋值过程和静态块的执行过程。
```

2.对象的创建过程

```
第一步：虚拟机遇到一个new指令时，会到常量池查是否这个类是否已经被加载、解析和初始化过。如果没有的话，需要进行一次类加载过程。
如果已经完成，那么虚拟机开始为对象分配空间。一般而言，这个对象会被优先分配到年轻代的Eden区。如果对象过大的话，也有可能会被分配到年老代。
如果Eden区没有足够的空间进行分配的话，会进行一次YGC。
```
